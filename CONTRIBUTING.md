# Contributing to Build

Build is an open-source, **learn-by-shipping-together** community for people building with AI agents. We collaborate in public, ship small, and teach each other what actually works. Contributions of all kinds are welcome — code, docs, design, research, marketing, and community.

## Our repositories

The [The-Last-Founder](https://github.com/The-Last-Founder) org currently has three active repos:

| Repo                                                               | What it is                                                                                                             |
| ------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------- |
| [**Build**](https://github.com/The-Last-Founder/Build)             | The central/main repo — community docs, methodology, tools, research, and coordination. **Start here.**                |
| [**Johnny**](https://github.com/The-Last-Founder/Johnny)           | A WhatsApp-native agent that helps build systems — our first pilot.                                                    |
| [**quest-board**](https://github.com/The-Last-Founder/quest-board) | A GitHub-native quest board where humans and AI agents propose, rank, build, and launch open-source projects together. |

## How we work

- **Learn by shipping together** — pick something small, ship it, share what you learned.
- **Do-ocracy** — the person who does the work gets autonomy on _how_ (see [GOVERNANCE.md](GOVERNANCE.md)).
- **Build in public** — default to open: issues, PRs, and discussion in the open.

## Finding something to work on

1. Browse the [high-priority board](https://github.com/orgs/The-Last-Founder/projects/1) or [all open issues](https://github.com/The-Last-Founder/Build/issues).
2. New to Git/GitHub? Start with [GIT.md](GIT.md).
3. Want to explore the toolset and how we work? See [TOOLS.md](TOOLS.md) and [METHODOLOGY.md](METHODOLOGY.md).

## Making a contribution

1. Fork the repo (or create a branch directly if you have write access).
2. Keep changes focused and small — smaller pull requests merge faster.
3. Open a pull request against `main` with a clear description of **what** changed and **why**.
4. Link the related issue if there is one.

**Pull requests welcome!**

## Spotted a problem or have an idea?

[Open an issue](https://github.com/The-Last-Founder/Build/issues/new) — bugs, improvements, and new ideas are all welcome.
